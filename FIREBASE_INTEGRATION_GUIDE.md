## 🚀 HOW IT WORKS

### Initial Load

1. App loads `firebase-config.js`
2. Firebase SDK initializes Firestore connection
3. Real-time listener attached to `products` collection
4. Products auto-update when data changes in Firestore
5. Cart loaded from Firestore `cart` collection

### Adding Products (Admin)

1. Admin opens panel (⚙️ icon)
2. Enters password: `admin123`
3. Fills product form
4. Clicks "Add Product"
5. Product sent to Firestore `products` collection
6. Real-time listener updates UI
7. Product appears in customer view instantly

### Shopping Cart

1. Customer adds product to cart
2. Cart data saved to Firestore `cart` collection
3. Cart persists across browser sessions
4. Multiple devices can sync (with user auth)

### Checkout

1. Customer clicks "Order via WhatsApp"
2. Order created in `orders` collection
3. WhatsApp message opens with order details
4. On WhatsApp payment, manual order update (optional)

## 🔐 SECURITY RULES EXPLANATION

### Public Read (Products)
```
allow read: if true;
```
- Anyone can view products
- No authentication required

### Admin Write (Products)
```
allow create, update, delete: if isAdmin();
```
- Only admins can modify products
- Non-admins get permission denied

### Cart (User Specific)
```
allow read, write: if true;
```
- Users can manage their own cart
- Data indexed by user session

### Orders (User Specific)
```
allow read, create: if true;
allow update: if isValidOrderUpdate();
```
- Users can view and create orders
- Can only update specific fields

## 🚨 TROUBLESHOOTING

### Issue: "Firebase is not defined"
**Solution**: 
- Clear browser cache
- Reload page
- Check that Firebase CDN scripts loaded (check Network tab in DevTools)

### Issue: "Permission denied" when adding products
**Solution**:
- You're not authenticated as admin
- Check FIRESTORE_RULES.txt for admin setup
- Use test mode rules for development (temporary)

### Issue: Products not loading
**Solution**:
- Check Firestore has `products` collection
- Check Security Rules allow public read
- Check browser console for errors
- Try localStorage fallback (no internet connection)

### Issue: Cart not saving
**Solution**:
- Check Firestore `cart` collection exists
- Check Security Rules allow cart writes
- Check browser localStorage as backup

### Issue: Images not loading
**Solution**:
- Verify image URLs are correct
- Check CORS settings in Cloud Storage (if using)
- Images must be publicly accessible
